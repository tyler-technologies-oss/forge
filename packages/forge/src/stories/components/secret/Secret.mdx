The secret component conceals sensitive content with a blur or dot mask, revealing it on user interaction.

<Canvas of={SecretStories.Demo} />

> **Note:** Concealed content is still present in the DOM and can be accessed in the page source. The secret component
> is useful for preventing shoulder surfing and accidental exposure, but it does not provide security against
> determined users. **The only way to truly secure sensitive information is to not send it to the client at all.**

## Named Groups

When multiple secrets share the same `name` attribute, they act as a radio group where only one can be revealed at a time.

<Canvas of={SecretStories.NamedGroup} />

## Block Layout

By default, the secret component displays inline with surrounding text. Setting it to `block` allows it to wrap larger pieces of content like paragraphs or images.

## Accessibility

- The component uses `role="group"` to semantically group the content and toggle button.
- The button uses `aria-expanded` to convey whether the secret content is visible.
- The button is associated with the group via `aria-controls`.
- When content is revealed, it's announced to screen readers using a live region.
- Hidden content is marked with the `inert` attribute to prevent keyboard navigation and screen reader access.
- The button is focusable and can be toggled with the keyboard (Enter or Space). When the secret is open pressing Escape closes it.
- The secret's content can be clicked to reveal it and focus the button.
