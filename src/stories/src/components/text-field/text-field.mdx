# Text field

The text field component supports many different variants, patterns, and configuration options. The `<forge-text-field>` element is a
block-level element by default and will take up 100% of the available width within its parent container.

> Masking functionality is not built-in to the component, it's expected that you'll use standard third-party mask libraries with it.
> If your team is using Angular, we recommend <a href="https://www.npmjs.com/package/ngx-mask">ngx-mask</a>; otherwise we recommend
> the <a href="https://www.npmjs.com/package/imask">imask package</a>; both available from npm.

## Properties/Attributes

> **Note:** When using `type="number"` on the `<input>` element, `maxlength` is ignored.  Use the `min`, `max`, and `step` properties to constrain the input.

<PropertyDef name="dense" type="boolean" defaultValue="false">

  Gets/sets the dense state.

</PropertyDef>

<PropertyDef name="floatLabelType" type="TextFieldFloatingLabelType" defaultValue="auto">

  The type of floating label to use.

  Valid values: `auto` (default), `always`.

</PropertyDef>

<PropertyDef name="shape" type="TextFieldShapeType" defaultValue="default">

  The outline shape type to use.

  Valid values: `default`, `rounded`.

</PropertyDef>

<PropertyDef name="invalid" type="boolean" defaultValue="false">

  Controls the invalid state. When set to `true`, the label and border will change to the error theme color.

> If you are using this for form validation, be sure to also update the property on the embedded `<input>` element.

</PropertyDef>

<PropertyDef name="required" type="boolean" defaultValue="false">

  Controls the presence of an asterisk after the label to signify that the field is required.

> If you are using this for form validation, be sure to also update the property on the embedded `<input>` element.

</PropertyDef>

<PropertyDef name="inset" type="boolean" defaultValue="false">

  Sets the style of the floating label to be inside the outline.

</PropertyDef>

## Slots

> A default (unnamed) slot exists, and it is intended that an `<input>` or `<textarea>` element will be placed in this slot.

| Name               | Values
| :------------------| :-----------------
| `label`            | Projects the label element into the correct location. If not specified on the `<label>` element then a slot attribute will be added automatically when connected to the DOM.
| `leading`          | Places content such as icons before the input.
| `default`          | The default (unnamed) slot is the text input. Projects the input into the correct location.
| `trailing`         | Places content such as icons after the input.
| `helper-text`      | Places content below the component with the proper typography.
| `addon-end`        | Places an action such as icon-button after the input.

</PageSection>

## CSS custom properties

| Name                              | Description
| :---------------------------------| :----------------
| `root`                            | The root element.
| `container`                       | The container layout element.
| `leading-container`               | The container around the `leading` slot.
| `label-input-container`           | The container element around the label and input slots.
| `addon-end-container`             | The container element around the addon-end slot.
