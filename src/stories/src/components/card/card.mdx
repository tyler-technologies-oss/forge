# Card

The card component is intended to be used as a layout design element to help group related content together into sections.

This a simple component that doesn't provide any functionality, but brings consistency through common default values. You are
free to place any content you would like into this component using its default slot.

</PageSection>

## Properties/Attributes

<PropertyDef name="raised" type="boolean" defaultValue="false">

Controls whether the card has elevation applied.

> Avoid using a raised card on another elevated surface.
  
</PropertyDef>

<PageSection>

## CSS custom properties

| Name                              | Description
| :---------------------------------| :----------------
| `--mdc-theme-surface`             | Controls the surface (background) color of the card.
| `--mdx-shape-medium`              | Controls the border radius of the card.
| `--forge-card-padding`            | Controls the `padding` of the container element.
| `--forge-card-overflow`           | Controls the `overflow` style of the card element.
| `--forge-card-height`             | Sets the `height` of the card element.
| `--forge-card-width`              | Sets the `width` of the card element.
| `--forge-theme-outline`      | Sets the border color of the card element when not raised.

</PageSection>

<PageSection>

## Accessibility

* The card component should not interfere with tab flow.
* Add a role attribute if this card has any applicable role that should be known to screen readers.
* Consider using the `group` role on the `<forge-card>` element if the content is related and separate from other surrounding items.
