# Floating Action Button (FAB)

Use floating action buttons to represent the primary action on a screen within an application. It's recommended to only use **one** floating action button
per screen.

Floating action buttons are static by default, and should be positioned properly above content by consumers. See below for an example
about how to position these elements in your application.

</PageSection>

## Example

Typically you will position these buttons manually on the screen, for example to apply a fixed position in the bottom-right you could use this CSS:

```css
.fab--bottom-right {
  position: fixed;
  bottom: 1rem;
  right: 1rem;
}
```

## Properties/Attributes

<PropertyDef name="theme" type="string" prop={false} defaultValue="&quot;secondary&quot;">

The theme type to apply. Allowed values: `primary`, `secondary` (default), `success`, `warning`, `danger`, `info`.
  
</PropertyDef>

<PropertyDef name="mini" type="boolean" defaultValue="false">

Displays the floating action button in its "mini" state.
  
</PropertyDef>

<PropertyDef name="extended" type="boolean" defaultValue="false">

Allows for a label to be used alongside the icon.
  
</PropertyDef>

<PropertyDef name="exited" type="boolean" defaultValue="false">

Controls the exited state. This animates the FAB in/out with a transition while leaving it in the DOM.
  
</PropertyDef>

</PageSection>

## Accessibility

* Buttons containing only icons should be given a meaningful label via `aria-label` or `aria-labelledby`.
* Avoid using capitalized text because screen readers will read the text character-by-character.
* Ensure the FAB can be reached by keyboard navigation.
* Ensure that there is a distinct visual cue when the FAB is in focus.
* Verify that there is sufficient contrast between the foreground text and background to meet WCAG requirements.
* Ensure that buttons placed above other content on the page have proper contrast ratio.
