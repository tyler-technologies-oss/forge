# Slider

Sliders allow users to make selections from a range of values.

Use sliders to enable users to select a value from a continuous or discrete range of values.

</PageSection>

<PageSection>

## Examples

### Default

<LiveDemo fullWidth layout="column" codeHtml={SliderDefaultHtml()}>
  <SliderDemo />
</LiveDemo>

### Range

You can also specify the component to use two handles by setting the range attribute:

<LiveDemo fullWidth layout="column" codeHtml={SliderRangeHtml()}>
  <SliderDemo range valueStart={30} valueEnd={70} />
</LiveDemo>

> **Note:** when using the range variant, you must set the start and end values separately with `valueStart` and `valueEnd`. 

### With tick marks

The following example will force the user to choose a discrete value from the range, with visual markers on the component:

<LiveDemo fullWidth layout="column" codeHtml={SliderTicksHtml()}>
  <SliderDemo tickmarks step={10} />
</LiveDemo>

## Accessibility

- Ensure that all of the controls that are accessible by a mouse are also accessible by keyboard.
- Ensure the slider controls are reachable by the tab key.
- Ensure each slider control can be updated by pressing the left-arrow and right-arrow keys.
- The slider element will receive the proper `aria-label` attribute.

> Be sure to set the `data-aria-label` attribute to provide a descriptive label for screen readers.
>
> For range mode, use the corresponding `data-aria-label-start` and `data-aria-label-end` attributes. These will apply different labels to the start and end handles.
>
> The `aria-label` attributes are propagated to internal elements, this is why they must to be provided as `data-*` attributes on the `<forge-slider>` host element.
