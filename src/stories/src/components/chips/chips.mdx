# Chips

The chip-set and chip components are used to display either grouped, or individual, interactive chip elements.

Chips can have one of the following types:

* Choice - Allows for selecting only one chip at a time
* Filter - Allows for selecting multiple chips.
* Action - A chip that acts as button for performing an action.
* Input - A chip that is "deleteable" and can be used to transform text into a chip element.
* Field - A chip that is intended to be used within a form field, such as the [chip-field component](?path=/story/components-chip-field--default).

> If you are trying to use chips as non-interactive elements, consider using a [badge component](?path=/story/components-badge--default) instead.

## Events

| Name                   | Description
| :----------------------| :----------------
| `forge-chip-select`      | Emits when an action, filter, or choice chip is clicked.
| `forge-chip-delete`      | Emits when the delete button on an input chip is clicked.

> **Note:** It's recommended that you take advantage of event propagation and listen for the event(s) on the `<forge-chip-set>` element.

</PageSection>

## Slots

### Chip set

| Name                           | Description
| :------------------------------| :----------------
| `default`                      | The default (unnamed) slot is where child `<forge-chip>` elements will be placed.

### Chips

| Name                           | Description
| :------------------------------| :----------------
| `default`                      | The default (unnamed) slot will render its content as the main text body.
| `leading`                      | The leading slot will render its content as the leading icon.
| `trailing`                     | The trailing slot will render its content as the trailing icon.

## Accessibility

- Verify that you can tab to each chip component.
  - Ensure that there is a visual cue that the chip component is active or inactive, and/or currently focused.
- Verify that pressing the space bar or enter key while focusing on a chip will toggle it in the same manner as if it had been clicked by a mouse.
- If a chip has a delete icon, ensure it can be focused and activated independently of the chip itself.
- Be sure to include a role attribute to indicate to screen readers what the chip is being used for.
- Verify that there is sufficient contrast between the foreground text and background to meet WCAG requirements.
- If color conveys important information, provide additional cues for users with color perception deficiencies.

</PageSection>

## Types

### ChipType

```ts
type ChipType =  'choice' | 'filter' | 'action' | 'input' | 'field';
```

### IChipSelectEventData

This is the type for the `detail` property on the `forge-chip-select` custom event.

```ts
interface IChipSelectEventData<T = any> {
  value: T;
  selected: boolean;
}
```

### IChipDeleteEventData

This is the type for the `detail` property on the `forge-chip-delete` custom event.

```ts
interface IChipDeleteEventData<T = any> {
  value: T;
}
```

### IChipState

```ts
interface IChipState {
  type: ChipType;
  disabled: boolean;
  dense: boolean;
}
```
