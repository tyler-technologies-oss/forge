# Linear progress

This component has a couple of variants that allows you to provide progress completion percentages, or a simple indicator that some indeterminate amount
of work is in progress. See the examples section below for information about each variant.

</PageSection>

---

<PageSection>

## Example

### Indeterminate

Use this variant when the progress completion percentage is unknown.

<LiveDemo fullWidth codeHtml={LinearProgressIndeterminateCodeHtml()}>
  <LinearProgressDemo />
</LiveDemo>

### Determinate

Use this variant when you know exactly how much progress has been made out of a total number of items.

<LiveDemo fullWidth codeHtml={LinearProgressDeterminateCodeHtml()}>
  <LinearProgressDemo determinate={false} />
</LiveDemo>

### Buffer

This variant is similar to determinate, except it allows you to show that the total amount of items is still being determined.

<LiveDemo fullWidth={true} codeHtml={LinearProgressBufferCodeHtml()}>
  <LinearProgressDemo progress={0.33} buffer={0.66} />
</LiveDemo>

</PageSection>

<PageSection>

## Properties/Attributes

<PropertyDef name="determinate" type="boolean" defaultValue="false">

Controls whether the component is using the determinate (`true`) or indeterminate (`false`) variant.

</PropertyDef>

<PropertyDef name="progress" type="number" defaultValue="0">

The progress amount (between `0` and `1`) of the indicator. Only valid in determinate state.

</PropertyDef>

<PropertyDef name="buffer" type="number" defaultValue="0">

The buffer amount (between `0` and `1`) of the indicator. Only valid in determinate state.

</PropertyDef>

<PropertyDef name="visible" type="boolean" defaultValue="false">

Controls the "open" state of the element by leaving it in the DOM, while toggling between visible and hidden with an animation.

This internally calls the `open()` and `close()` methods defined below.

</PropertyDef>

<PropertyDef name="progressbar-aria-label" type="string" defaultValue="''" prop={false}>

Sets the `aria-label` attribute on the internal `progressbar` element.

> **Note:** You should **always** set this attribute for compliance with accessibility requirements.

</PropertyDef>

</PageSection>

<PageSection>

## Methods

<MethodDef name="open(): void;">

Shows the component if it is in the closed state.

</MethodDef>

<MethodDef name="close(): void;">

Closes the component if it is in the open state.

</MethodDef>

</PageSection>

## CSS parts

| Name                                       | Description
| :------------------------------------------| :----------------
| `root`                                     | The root container element.
| `buffer`                                   | The buffer bar container element.
| `buffer-bar`                               | The buffer bar element.
| `buffer-dots`                              | The buffer bar animating dots element.
| `progress-bar`                             | The progress bar container element.
| `progress-bar-inner`                       | The progress bar progress element.
| `secondary-bar`                            | The secondary bar container element.
| `secondary-bar-inner`                      | The secondary bar progress element.

</PageSection>

<PageSection>

## CSS custom properties

| Name                                                      | Description
| :---------------------------------------------------------| :----------------
| `--forge-theme-tertiary`                                  | Controls the inner color of the progress bar.
| `--forge-linear-progress-border-radius`                   | Controls the `border-radius` style of the progress bar for when it is placed within a container that has rounded corners.
| `--forge-linear-progress-color`                           | Controls the color of the progress bar.
| `--forge-linear-progress-theme-buffer`                    | Controls the color of the buffer bar.
| `--forge-linear-progress-theme-buffer-dots-filter`        | Controls the color of the buffer dots filter.

</PageSection>
