# Quantity field

The quantity field component wraps a decrement button, numeric input, and an increment button. Clicking on either button will increment or decrement the value of the numeric input.

The quantity field is used for fine-tuning numeric values with a predictable range and step.  
e.g. If the default value is 1 and you want the user to potentially increment up to 5.

It should not be used for unpredictable ranges or awkward steps.  
e.g. If the default value is expected to change within a range of 0-1000 and the only step that makes sense is 1... this is likely not the component you want to use.

If the user is unlikely to use the increment and decrement buttons to get to the value they will want, instead consider just using a [text-field](?path=/story/components-text-field--default) with an input type of "number".

## Properties/Attributes

<PropertyDef name="invalid" type="boolean" defaultValue="false">

Controls the invalid state. When set to `true`, the label and border will change to the error theme color.
  
</PropertyDef>

<PropertyDef name="required" type="boolean" defaultValue="false">

Controls the presence of an asterisk after the label to signify that the field is required.

</PropertyDef>

</PageSection>

<PageSection>

## Slots

| Name                           | Description
| :------------------------------| :----------------
| `label`                        | Projects the label element into the correct location with the proper typography.
| `decrement-button`             | Projects the button used for decrementing the input's value to the left of the input.
| `default`                      | The default (unnamed) slot is the numeric input. Projects the input into the correct location.
| `increment-button`             | Projects the button used for incrementing the input's value to the right of the input.
| `helper-text`                  | Places content below the component with the proper typography.

</PageSection>

<PageSection>

## Events

This component doesn't dispatch any events itself, but the `<input>` element will dispatch both an `input` and `change` event when the value changes
after clicking either increment or decrement button.

</PageSection>
