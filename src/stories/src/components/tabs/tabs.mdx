# Tabs

Tabs organize content across different screens and views.

Use tabs to group content into helpful categories. Tabs are typically placed above the content they relate to.
Tabs can be used to navigate between screens, or to group related content within a screen.

This component does **not** handle swapping out tab panels for you, but it will allow for you respond to the tab
index changing to then hide/show the correct panel in your content.

Tabs can easily be paired with [view switcher component](?path=/story/components-view-switcher--default) to switch between views on a page.

## Tab Bar Properties/Attributes

<PropertyDef name="activeTab" type="number | null | undefined" defaultValue="undefined">

  Controls the active tab index.

</PropertyDef>

<PropertyDef name="vertical" type="boolean" defaultValue="false">

Controls whether the tabs are rendered vertically or not.

</PropertyDef>

<PropertyDef name="clustered" type="boolean" defaultValue="false">

Controls whether the tabs are rendered in a cluster (collapsed to minimum width) or not (stretched to fill available space).

</PropertyDef>

<PropertyDef name="stacked" type="boolean" defaultValue="false">

Controls whether the tab content is stacked vertically or not.

</PropertyDef>

<PropertyDef name="secondary" type="boolean" defaultValue="false">

Controls whether the tabs are rendered as secondary tabs or not.

Use secondary tabs when there is a need for multiple levels of tabs on the same page.

</PropertyDef>

<PropertyDef name="inverted" type="boolean" defaultValue="false">

Controls whether the tab indicators are rendered on the opposite side of the tabs.

</PropertyDef>

<PropertyDef name="autoActivate" type="boolean" defaultValue="false">

Controls whether tabs are auto-activated when using arrow keys.

Keys which auto activate: `ArrowLeft`, `ArrowRight`, `Home`, `End`, as well as `ArrowUp` and `ArrowDown` when in vertical mode.

</PropertyDef>

<PropertyDef name="scrollButtons" type="boolean" defaultValue="false">

Controls whether scroll buttons are rendered or not when the tab container is scrollable.

</PropertyDef>

</PageSection>

## Keyboard shortcuts

| Name                                | Description
| :-----------------------------------| :----------------
| `End`                               | Focus end tab.
| `Space/Enter`                       | Activate the focused tab.
| **Horizontal**                      |
| `ArrowLeft`                         | Focus previous tab.
| `ArrowRight`                        | Focus next tab.
| **Vertical**                        |
| `ArrowUp`                           | Focus previous tab.
| `ArrowDown`                         | Focus next tab.

> **Note:** If `autoActivate` is enabled, using the arrow keys will also activate the tab when focused.

## Accessibility

- Ensure that all of the controls that are accessible by a mouse are also accessible by keyboard.
- Ensure the controls are reachable by the tab key.
- Ensure each control can be updated or activated by the keyboard.
- Ensure that only the selected tab content is reachable by keyboard.
- Ensure a tab is always active at any given time.

It's expected that you will need to provide the `tabpanel` role to your elements to indicate to assistive technologies
which element is responsible for tab content. Each tab should contain an `aria-controls` property identifying
its corresponding tab content. In addition, each tab content element should be labelled by a tab:

```html
<forge-tab-bar>
  <forge-tab aria-controls="panel-1" id="tab-1">Tab 1</forge-tab>
  <forge-tab aria-controls="panel-2" id="tab-2">Tab 2</forge-tab>
</forge-tab-bar>
<div role="tabpanel" aria-labelledby="tab-1" id="panel-1">Content 1</div>
<div role="tabpanel" aria-labelledby="tab-2" id="panel-2">Content 2</div>
```

> **Note:** the `<forge-tab>` elements receive the `role="tab"` automatically.
