## Textarea

Text fields allow for providing a `<textarea>` element in place of an `<input>` element for multi-line text input.

<Canvas of={TextFieldStories.Textarea} />

## Label Position

The text field supports a `labelPosition` property/attribute to control the position of the label. The default value is `"inset"`
where the label is positioned inside the text field, but it can also be set to `"block-start"` or "inline-start" to position the
label above or to the left of the text field respectively.

### Block Start

This variant positions the label above the text field.

<Canvas of={TextFieldStories.LabelAbove} />

### Inline Start

This variant positions the label to the left of the text field.

<Canvas of={TextFieldStories.LabelInline} />

> **Note:** The `labelPosition` property is available via [global configuration](?path=/docs/getting-started-global-configuration--docs) if
> you want to set adjust the default value for all text fields in your application.

## Accessibility

- Ensure that if you are not using a `<label>` element, you provide an `aria-label` or `aria-labelledby` attribute to the `<input>` element.

### Required Field Markup

To ensure proper validation and accessibility, apply the `required` attribute to both the `<forge-text-field>` component and its nested `<input>` element:

- The `required` attribute on `<forge-text-field>` is **presentational**, used to display the required field indicator in the UI.
- The `required` attribute on the nested `<input>` is **semantic**, enabling native form validation and improving accessibility support.

**Example:**

```html
<forge-text-field required>
  <input required />
</forge-text-field>
```

## CSS-Only

The text-field component is also available as a CSS-only component. This is a variant of the "field" component.

See the [field documentation](?path=/docs/components-field--docs#css-only) for more information on how to create a CSS-only text-field.
